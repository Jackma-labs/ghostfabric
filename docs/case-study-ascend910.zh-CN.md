# 案例研究：GhostFabric 如何提升老旧昇腾 910 集群的可用性与专家能力

## 背景

很多团队手里依然有已经部署好的 `Ascend 910` 集群。硬件并没有消失，但围绕它的软件层经常逐渐失效：

- 推理服务栈逐渐漂移
- 组件之间兼容性变差
- 单次生成不稳定
- 检索能力弱，模型无法成为真正可用的专家
- Agent 运行时不是太脆，就是太危险

最后的结果很常见：算力还在，但生产力不在。

GhostFabric 要解决的正是这个缺口。

## 问题不在“卡还能不能算”，而在“系统还能不能用”

老旧加速卡环境很少是因为芯片彻底不可用才失去价值。更常见的情况是整套系统的运维成本越来越高：

- 推理质量不稳定
- 接口和兼容层不断积累技术债
- 文档和运维知识碎片化
- 工程师不再信任自动化辅助
- 老集群被当成“死库存”，而不是软件优化对象

对于老旧 `Ascend 910` 部署，瓶颈往往不是纯算力，而是系统编排质量。

## GhostFabric 改变了什么

GhostFabric 在四层做增强：

### 1. 置信度感知解码

GhostFabric 不把一次生成当成真相，而是用 DeepConf 风格的选择流程：

- 生成多条候选推理轨迹
- 用 token 级置信度打分
- 选择更强的结果
- 当最优结果和次优结果拉开足够差距时提前停止

这对 `Qwen2.5` 这类“有能力，但单次稳定性不够”的模型尤其有用。

### 2. 检索增强

老集群的知识通常散落在很多地方：

- 部署说明
- 启动脚本
- 运维手册
- 故障排查记录
- 历史 workaround

GhostFabric 的做法不是让模型“猜”，而是把这些材料变成真正可检索、可评测、可复用的专家资产。

### 3. 兼容层和路由层

真实线上问题很多不在模型本身，而在 glue code：

- OpenAI 兼容层
- tool-calling 兼容
- 后端路由
- 流式输出标准化
- 请求清洗

GhostFabric 把这些问题当作系统工程的一部分，而不是假设只要有个模型接口就够了。

### 4. 受限执行

一个真正有用的基础设施专家不能只会解释，它还需要在边界内执行：

- 读文件
- 搜代码
- 看日志
- 执行白名单命令
- 提议或执行受限修改

对老旧集群尤其如此，因为危险自动化的代价往往高于低效人工操作。

## 为什么这对 Ascend 910 特别重要

对于老旧 `Ascend 910` 环境，目标不是把旧硬件说成新硬件，而是通过改善硬件周围的软件路径，把原本沉睡的算力重新变成可用算力。

GhostFabric 的价值主要体现在三点：

### 更高可用性

如果系统能更稳定地解释故障、定位配置问题、给出有证据支撑的操作答案，那么现有集群就更容易重新投入使用。

### 更强工程杠杆

如果工程师能通过一个统一的专家层同时查询代码、配置、文档、日志和运行手册，维护成本就会明显下降。

### 更高有效性能

如果系统减少了错误生成、路由问题和人工排查时间损耗，那么即便不更换物理加速卡，整体产出仍然会提升。

核心观点是：

> 真正可用的算力，不只取决于硬件容量，也取决于软件纪律。

## 一个现实可行的运行闭环

围绕老旧 `Ascend 910` 集群，GhostFabric 可以这样落地：

1. 收集部署文档、启动说明、脚本和排障记录
2. 从真实运维问题中构建严格评测集
3. 先把检索做好，再谈专家能力
4. 对单次生成不稳定的问题启用 DeepConf
5. 开启受限 Agent 动作，用于巡检和有限修复
6. 对高风险操作始终保留人工审批

这样得到的是一个“能帮你真正干活”的专家层，而不是一个看起来聪明但不可托付的自动化幻觉。

## 应该怎么衡量这件事

围绕 `Ascend 910` 的案例研究，建议至少衡量：

- 严格问答通过率
- 弱证据场景下的保守拒答准确率
- DeepConf 相比 one-shot 的延迟代价
- 受限执行下的 Agent 任务完成率
- 人工排障耗时是否下降

重点不是好看的宣传数字，而是：这些老旧基础设施是否真的变得更容易运维、更值得继续在线。

## GhostFabric 不声称什么

GhostFabric 并不声称：

- 老旧 `Ascend 910` 能等价于新硬件
- 只靠 DeepConf 就能修复弱检索
- Agent 应该拥有无限权限
- 基础设施自动化可以绕过人工审批

GhostFabric 的主张更窄，也更可信：

它可以让老旧加速器环境变得更可用、更可解释，也更有生产价值。

## 结论

对于老旧 `Ascend 910` 集群，重新获得价值的现实路径既不是情怀，也不是宣传，而是软件工程：

- 更好的解码
- 更好的检索
- 更好的兼容层
- 更清晰的安全边界

这正是 GhostFabric 试图做的事情。
